# Building Relay

A premium **native** macOS Messenger client (SwiftUI, Liquid Glass) backed by a Go
service (`relay-helper`) that speaks Meta's real protocol — **not** a web wrapper.

**Requirements:** **Xcode 26+** to build (the UI uses the macOS 26 SDK with per-OS
availability fallbacks), a **Go** toolchain, and `xcodegen`. The built app **runs on
macOS 13 Ventura or later**, Intel or Apple Silicon. The project is defined by
`project.yml` (XcodeGen), so source files can be added without hand-editing the
`.xcodeproj`. Set `DEVELOPMENT_TEAM` in `project.yml` to your own Apple team id.

## Build, sign, install & launch (the one command you need)
```bash
cd Relay
scripts/run-native.sh
```
This builds the **universal** (Intel + Apple Silicon) Go helper, regenerates the
project, builds `Relay`, bundles the helper inside the app, signs it (Apple
Development cert + hardened runtime + `RelayNative/RelayNative.entitlements`),
installs to `/Applications`, and launches. A stable code identity means granted
**Notification**/**Microphone** permissions persist across rebuilds.

## Open in Xcode (optional)
```bash
brew install xcodegen          # if not already installed
xcodegen generate              # produces Relay.xcodeproj from project.yml
open Relay.xcodeproj
```
Select the **Relay** scheme → Run. Re-run `xcodegen generate` whenever files are
added/removed.

## Notes
- Not sandboxed — direct distribution only, never the Mac App Store.
- Deployment target **macOS 13.0**; universal binary. Newer-OS features (Liquid Glass
  on 26, Translation/scroll on 15) are gated via `#available` — see `RelayNative/Glass.swift`.
- Entitlements: hardened-runtime JIT flags + `com.apple.security.device.audio-input`
  (microphone, for voice notes).
- The app icon is generated by `scripts/make-icon.py` into
  `RelayNative/Assets.xcassets/AppIcon.appiconset` (re-run it to regenerate).

## Architecture
- **`RelayNative/`** — the SwiftUI app (folder name predates the rename to "Relay"):
  - `RelayApp.swift` — app entry, glass window background, Esc/paste handling.
  - `Backend.swift` — `RelayStore` (state), `HelperClient` (talks to the Go helper over
    a pipe), `VoiceRecorder`, persistence.
  - `Views.swift` — sidebar, thread view, bubbles, media, reaction picker, composer.
  - `Glass.swift` — per-OS availability shims (Liquid Glass ↔ material, key handling, etc.).
  - `MessageStore.swift` — local SQLite history + full-text search.
- **`relay-helper/`** — the Go backend:
  - `main.go` — command loop + Lightspeed (`messagix`) handling (non-E2EE, groups).
  - `e2ee.go` — whatsmeow handling (E2EE 1:1 chats — the default path now).
- **`thirdparty/mautrix-meta`** — vendored fork of `messagix` (AGPL-3.0).
